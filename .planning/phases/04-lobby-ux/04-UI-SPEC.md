## Spacing Scale

Base unit: 8px. Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicators |
| sm | 8px | Gap between game picker tiles; gap between QR card inner elements |
| md | 12px | Internal padding on compact elements (QR card body padding) |
| lg | 16px | Body horizontal padding; button padding; input padding |
| xl | 24px | Card padding; gap between sections |
| 2xl | 32px | Gap between landing page sections; card grid gap |
| 3xl | 48px | Page-level top/bottom margin; max-width container margin |

Exceptions:
- Game picker tiles: min-height 80px (10 ├Ś 8px) to provide a generous tap target
- QR code image block: fixed to 160px ├Ś 160px (native SVG at this rendered size)
- Room code display: letter-spacing 0.2em (carries over from existing spec)

Source: DESIGN.md spacing scale (4, 8, 12, 16, 24, 32, 48, 64) and existing `LOBBY_HTML` patterns (48px margin, 16px padding).

## Typography

Both pages use Inter exclusively for functional UI. Achafont appears only in game name display blocks on the room page title (BOXING / DANCE) rendered at `--type-hero` scale.

| Role | Font | Size | Weight | Letter-spacing | Line Height | Usage |
|------|------|------|--------|----------------|-------------|-------|
| Page title / SPECTRE | Inter | 28px (`--type-hero` capped) | 900 | 0.12em | 1.1 | SPECTRE header on landing page |
| Section label / subtitle | Inter | 12px (`--type-small`) | 700 | 0.08em | 1.3 | Section dividers ("Join a Room"), card role labels (PLAYER 1) |
| Game tile label | Inter | 20px | 900 | 0.1em | 1.1 | BOXING / DANCE labels inside picker tiles |
| Body / CTA button | Inter | 16px (`--type-body`) | 400 body, 800 buttons | 0.08em buttons, 0 body | 1.5 body | Button text, input text, URL links, copy button |
| Room code display | Inter | 32px | 900 | 0.2em | 1.0 | Prominent room code on `/rooms/{code}` page header |
| URL link text | Inter | 12px (`--type-small`) | 700 | 0.04em | 1.4 | Prefilled URL text under each QR code |

Rules (from DESIGN.md):
- All labels uppercase. Letter-spacing 0.08em minimum on any uppercase text.
- No font-size below 12px anywhere.
- Achafont at `clamp(32px, 6vw, 88px)` weight 900 for game-name display on room page only ŌĆö not for functional UI.

## Color

All values are OKLCH from DESIGN.md. Never pure black or white.

| Role | Token | OKLCH | Hex approx | Usage |
|------|-------|-------|------------|-------|
| Dominant (60%) | `--bg-deep` | `oklch(7% 0.008 22)` | `#0c0809` | Page background, full-screen canvas |
| Secondary (30%) | `--bg-mid` + `--bg-surface` | `oklch(11% 0.009 22)` / `oklch(17% 0.01 22)` | `#141010` / `#201a1a` | QR cards, game picker tiles, join input background |
| Accent P1 (10%) | `--accent` | `oklch(44% 0.22 22)` | `#8b1a1a` | P1 card border, selected game-tile border, Create Room primary button border/bg tint |
| Accent P2 | `--accent-p2` | `oklch(50% 0.18 250)` | `#1a3a7a` | P2 card border |
| Gold | `--gold` | `oklch(78% 0.11 85)` | `#c8a84b` | QR card structural borders (Level 1 elevation), Overlay card border |
| Text primary | `--text-primary` | `oklch(95% 0.008 85)` | `#f5efe4` | Headings, button text, room code, body |
| Text secondary | `--text-secondary` | `oklch(65% 0.008 85)` | `#9c9180` | Subtitle labels, URL link text |
| Text dim | `--text-dim` | `oklch(38% 0.006 85)` | `#524a42` | Default button borders, placeholder text, disabled states |
| Destructive | `--accent-bright` tinted | `oklch(60% 0.25 22)` at 15% bg | `rgba(212, 42, 42, 0.15)` border | Error state only (room not found, server error) |

Accent reserved for:
- `--accent` (crimson): P1 QR card border, selected game tile border highlight, Create Room primary button (border + 15% bg fill), focus ring on code input
- `--accent-p2` (steel): P2 QR card border only
- `--gold`: QR card structural borders (all three cards share Level 1 gold border as HUD elevation spec), Overlay card border

Rules from DESIGN.md:
- Gold is reserved for HUD structural borders. QR cards are structural, so gold border applies.
- Crimson belongs to P1 and functional moments. Not decorative.
- No neon, no glassmorphism, no glow on non-dramatic elements.

60/30/10 split:
- 60%: `--bg-deep` canvas
- 30%: `--bg-surface` cards + `--bg-mid` tile hover states
- 10%: accent touches (P1 crimson, P2 steel, gold borders)

## Component Specifications

1. **SPECTRE header block**

   - Tagline below: `real punches. real fights.` ŌĆö Inter 700 12px, letter-spacing 0.08em, uppercase, `--text-secondary`
   - Margin-bottom: 32px

2. **Game picker section**

   - Two tiles in a flex row, gap 8px, each tile: flex 1, min-height 80px

   - Tile label: Inter 900 20px uppercase letter-spacing 0.1em, `--text-primary`, centered

   - Tile active: transform scale(0.97) 80ms ease-out

3. **Create Room button**

   - Margin-bottom: 32px

4. **Separator**
   - Horizontal rule: 1px `--text-dim` at 40% opacity, or 24px spacer with centered label `or` in `--text-dim` 12px

5. **Join by code section**
   - Section label: `Join a Room` ŌĆö Inter 700 12px uppercase letter-spacing 0.08em, `--text-secondary`, margin-bottom 12px
   - Row: text input + Join button, gap 8px

### Error States

- **Join empty input**: Prevent navigation; no visual error ŌĆö button stays disabled while input is empty.

## Interaction Contracts

### Landing Page Interactions

| Trigger | Behavior |
|---------|----------|
| Page load | No game selected; Create Room disabled (opacity 0.5) |
| Click game tile (unselected) | Tile gains selected border + bg; other tile returns to default; Create Room becomes enabled |
| Click game tile (already selected) | No-op ŌĆö tile stays selected (selection required, not a toggle) |
| Click Create Room (disabled) | No-op |
| Click Create Room (enabled) | Button shows `Creating...` + disabled; `POST /rooms?game={type}`; on success navigate to `/rooms/{code}`; on error show error message below button |
| Type in room code input | Input uppercases value; Join button enables when input is non-empty |
| Click Join (with code) | Navigate to `/mobile?room={CODE}&server={ORIGIN}` ŌĆö standard `<a>` or `window.location.href` |
| Click Join (empty input) | No-op (button disabled) |

### Room Page Interactions

| Trigger | Behavior |
|---------|----------|
| Page load | Three QR cards fully rendered; QR SVGs embedded inline |
| Click URL link | Opens link in new tab (`target="_blank"`) |
| Click Copy Link | Calls `navigator.clipboard.writeText(url)`; button shows `Copied!` for 2000ms then reverts |
| Click back link | Navigates to `/` (standard anchor) |

### Responsive Behavior

- Landing page: max-width 480px, centered. Game picker tiles maintain 50/50 split down to 320px minimum.
- Room page: 3-column grid collapses to 1-column at viewport width <600px; card order stays P1 ŌåÆ P2 ŌåÆ Overlay top-to-bottom.
- QR code SVGs: fixed 160px ├Ś 160px at all breakpoints (do not scale).
- All touch targets: min 44px ├Ś 44px (buttons and tiles already meet this via min-height 52px and 80px respectively).
