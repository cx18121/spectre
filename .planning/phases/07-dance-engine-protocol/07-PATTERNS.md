/// Returns false if this game does not require pose calibration before match start.

**Object safety rule:** All new methods satisfy the same rules as existing ones — no `async fn`, no `-> Self`, no generic params, `&self` receiver.

## Shared Patterns

### Trait Default Method Pattern
**Source:** `engine/plugin-trait/src/lib.rs` lines 257–323
**Apply to:** All three new `GamePlugin` methods (`game_type`, `requires_calibration`, `spectator_snapshot`)

Rules that ALL new methods must satisfy:
1. `&self` receiver (not `&mut self`)
2. Synchronous — no `async fn`, no `await`, no blocking calls
3. No generic type parameters on the method signature
4. No `-> Self` return type
5. Default implementations provide sensible no-op or conservative value

### Plugin State Downcast Pattern
**Source:** `engine/dance-plugin/src/lib.rs` lines 80–81 and 187–188
**Apply to:** `DancePlugin::spectator_snapshot`

```rust
// Immutable downcast (for snapshot):
let s = state.downcast_ref::<DanceState>()
    .expect("dance plugin: spectator_snapshot type mismatch");

// Mutable downcast (for on_tick, on_round_reset):
let s = state.downcast_mut::<DanceState>()
    .expect("dance plugin: state type mismatch");
```

### Protocol Struct with ts-rs Export
**Source:** `engine/engine-core/src/protocol.rs` lines 165–175 (`MsgRoundStart`)
**Apply to:** `MsgDanceBeat`, `MsgDanceScore` (new), `MsgJoined` (extend)

Every outbound protocol struct follows this exact pattern:
1. `fn default_type_<name>() -> String { "<wire_type>".to_string() }`
2. `#[derive(Serialize, Deserialize, TS, Clone, Debug)]`
3. `#[ts(export)]`
4. `#[serde(rename = "type", default = "default_type_<name>")]` on `msg_type: String`

### WebSocket Message Send Pattern
