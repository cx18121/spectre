# Phase 9 — UI Design Contract: Dance Frontend

> Visual and interaction contract for the dance game overlay HUD, target pose skeleton,
> dance match end screen, and mobile calibration skip.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Spacing Scale

Base unit: 8px. Scale in use across existing overlay and mobile:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, win-dot gap (6px exception below) |
| sm | 8px | Compact internal padding, HUD element gaps |
| md | 16px | Default HUD padding (`.hud-band` left/right) |
| lg | 24px | Section padding, match-end card padding |
| xl | 32px | Layout gaps between major HUD regions |
| 2xl | 48px | Lobby top margin, major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Win-dot gap: 6px (existing `.win-dots` — not used in dance HUD, noted for boxing parity)
- HUD band: padding 16px 20px 14px top (existing `.hud-band` spec — reused unchanged)
- Beat indicator bar height: 4px (fixed, not on the spacing scale — functional size only)
- Match-end stats grid: 20px column padding (existing `.match-stats-col` — inherited)

Source: `DESIGN.md §Spacing`, `overlay/src/index.css`.

## Typography

All from the existing type scale in `DESIGN.md §Typography`. No new tokens needed for Phase 9.

| Role | Token | Size | Weight | Letter-spacing | Line Height | Use |
|------|-------|------|--------|----------------|-------------|-----|
| Display | `--type-display` | `clamp(72px, 15vw, 200px)` | 950 | 0 | Round flash headline (Achafont) |
| Hero | `--type-hero` | `clamp(32px, 6vw, 88px)` | 900 | 0 | Match end title (Achafont) |
| Score / Timer | `--type-hud-timer` | 36px | 900 | 0.02em | Dance score numbers (Inter) — reuses existing timer token |
| Label | `--type-label` | 12px | 800 | 0.1em | HUD labels, beat count label, "WINNER" label |
| Body | `--type-body` | 16px | 400 | 0 | Mobile screen body text |
| Small | `--type-small` | 12px | 700 | 0.06em | Status pills, sublabels |

Dance-specific typography details:

- **Dance score numbers (P1, P2):** Inter 900, 36px, tabular-nums, `--text-primary`. Matches `--type-hud-timer` for visual parity with the boxing timer. Format: one decimal precision — e.g. `11.7`. Zero-state: `0.0`.
- **Beat count label:** Inter 700, 12px, letter-spacing 0.1em, uppercase, `--text-secondary`. Format: `N / total_beats` e.g. `4 / 16`.
- **"vs" separator (HUD score row):** Inter 700, 12px, letter-spacing 0.1em, uppercase, `--text-secondary`, centred.
- **Dance match end — winner score:** Inter 900, `clamp(48px, 8vw, 96px)`, winner's accent color.
- **Dance match end — loser score:** Inter 900, `clamp(48px, 8vw, 96px)`, `--text-secondary`.
- **Dance match end — "WINNER" label:** Inter 900, 12px, letter-spacing 0.14em, uppercase, winner's accent color.
- **Dance match end — player labels ("P1" / "P2"):** Inter 900, 12px, letter-spacing 0.1em, `--text-secondary`.
- **Round end subscores:** Inter 700, 18px, `--text-secondary`. Format: `P1: 11.7  P2: 9.2`.
- **Round flash headline (dance):** Achafont 950, `clamp(52px, 10vw, 130px)` — inherited from `.round-flash`. Copy changes only; style unchanged.

Typeface rules:
- Achafont: round flash display only. Never used for functional information (scores, labels, copy).
- Inter: everything else. No exceptions in this phase.
- No font-size below 12px anywhere.
- All HUD labels are uppercase with minimum letter-spacing 0.08em.

Source: `DESIGN.md §Typography`, `08-CONTEXT.md D-01`, `08-CONTEXT.md D-04`.

| Text Primary | `--text-primary` | `oklch(95% 0.008 85)` | `#f5efe4` | Score numbers, match-end body text |

| Text Dim | `--text-dim` | `oklch(38% 0.006 85)` | `#524a42` | Target pose skeleton keypoints and bone lines (at 40% opacity) |

Accent reserved for:
- `--accent` / `--accent-p2`: match-end "WINNER" label color (winner's side only); HUD game-type routing does not use accent for dance-mode decoration
- `--accent-bright` / `--accent-p2-bright`: match-end winner score large number color
- Gold (`--gold`): HUD structural borders only — never text, never buttons

**Skeleton color rule:** Target pose skeleton uses `--text-dim` at 40% opacity for both keypoints (filled circles) and bone lines (stroked). Never `--text-secondary` — `--text-dim` is dimmer and does not compete with the beat indicator bar. Source: `DESIGN.md §Dance Game — Target Pose Skeleton`, `DDES-03`.

**Dance HUD rule:** No crimson accent decoration in the dance HUD band. The HUD band is neutral (`--bg-mid` gradient, `--gold` borders). Player accent colors appear only on match end to mark the winner.

## Component Inventory

These components are new or modified in Phase 9. Each maps to a plan.

### New: `DanceHud` (overlay — plan 09-02)

Sibling to `HudLayer`. Renders inside the same `.hud-layer` / `.hud-band` shell.

**Row 1 — Names and Beat Indicator:**
Layout: `display: grid; grid-template-columns: 1fr auto 1fr; align-items: end; gap: 14px` — same as boxing `.hud-names`.

- P1 column: "P1" — Inter 900, 12px, letter-spacing 0.1em, uppercase, `--text-primary`. Left-aligned. No win dots.
- P2 column: "P2" — same spec, right-aligned. No win dots.
- Centre column (beat indicator): width 40% of `.hud-band` total width.

**Beat Indicator (centre column of Row 1):**

Structure (top to bottom):
1. Beat count label: `N / total_beats` — Inter 700, 12px, letter-spacing 0.1em, uppercase, `--text-secondary`. Centred.
2. Draining bar: full-width within centre column, height 4px.
   - Track: `background: var(--bg-surface)`, no border, no border-radius.
   - Fill: `background: var(--text-secondary)`.
   - Reset: `transition: width 0ms` — hard snap to 100% on each `dance_beat` event.
   - Drain: `transition: width {beat_duration_ms}ms linear` — drains to 0% by next beat boundary.
   - No border-radius. No glow. No box-shadow.

**Row 2 — Scores:**
Layout: `display: flex; flex-direction: row; align-items: baseline; justify-content: space-between`.

- P1 score: Inter 900, 36px, tabular-nums, `--text-primary`, left-aligned within P1 column.
- "vs" separator: Inter 700, 12px, letter-spacing 0.1em, uppercase, `--text-secondary`, centred.
- P2 score: Inter 900, 36px, tabular-nums, `--text-primary`, right-aligned within P2 column.
- No bar, no fill, no color coding by score value.

HUD band structural rules unchanged: `--bg-mid` gradient, Level 1 border (`1px --gold 20% opacity, inset 0 1px 0 rgba(255,255,255,0.04)`). Reuses `.hud-band`, `.hud-layer` CSS classes.

### Modified: `PixiCanvas` — Target Pose Skeleton (plan 09-03)

New Pixi.js `Graphics` object drawn from `MsgDanceBeat.target_pose`.

**Position and scale:**
- Horizontal: centred in the canvas (midpoint between the two player silhouette zones).
- Vertical: same Y-origin as live player silhouettes (human scale, hip-centred Y-up normalised coordinates mapped to canvas height).
- Scale: matches player silhouettes — ghost body appears life-size alongside the live players.
- Does not overlap either player's canvas zone.

**Visual style:**
- Keypoints: filled circles, color `--text-dim` (`oklch(38% 0.006 85)`), Pixi alpha 0.4.
- Bone lines: stroked lines using `CONNECTIONS` from `overlay/src/lib/skeleton.ts`, same color, Pixi alpha 0.4.
- Visibility gate: only draw keypoints (and their bones) where `visibility >= 0.5`.
- No fill gradient, no accent color, no glow.

**Beat-swap transition:**
1. On `dance_beat` event — fade out: alpha 0.4 → 0.0 over 150ms, easing ease-out-quart. Applied to the whole `Graphics` object alpha, not per-keypoint.
2. After fade-out completes (150ms): redraw `Graphics` object from new `target_pose` keypoints.
3. Fade in: alpha 0.0 → 0.4 over 150ms, easing ease-in.
4. Total swap time: 300ms.
5. Animation driver: gsap (already in overlay) or `requestAnimationFrame` ticker. Must not block the Pixi render loop.

### Modified: `RoundOverlay` — Dance copy (plan 09-04)

Reuses `.round-flash` component. Only text content changes.

Dance round end copy:
- Winner determined: `ROUND N — P1 LEADS` or `ROUND N — P2 LEADS`
- Tie: `ROUND N — TIED`
- Scores below headline: Inter 700, 18px, `--text-secondary`. Format: `P1: 11.7  P2: 9.2`. P1 left, P2 right, same horizontal layout as HUD score row.
- Visibility window: 2200ms (unchanged from boxing).
- Motion spec: unchanged (`.round-flash` `@keyframes round-flash` animation).

Prohibited: "WINS", "KO", "TIME", any damage or HP reference.

### New: Dance Match End (plan 09-04)

New content within the existing `.match-end-overlay` shell. Same background treatment: `oklch(5% 0.005 22 / 0.78)`, Level 3 elevation.

Layout (top to bottom, centred):
1. "WINNER" label — Inter 900, 12px, letter-spacing 0.14em, uppercase. Color: `--accent-bright` if P1 wins, `--accent-p2-bright` if P2 wins. Omit / replace with "TIED" (no accent highlight) on tie.
2. Score row: two large score numbers side by side with "vs" separator.
   - Winner score: Inter 900, `clamp(48px, 8vw, 96px)`, tabular-nums. Color: `--accent-bright` (P1) or `--accent-p2-bright` (P2). Format: one decimal.
   - "vs" separator: Inter 700, 16px, `--text-secondary`, centred.
   - Loser score: Inter 900, `clamp(48px, 8vw, 96px)`, tabular-nums, `--text-secondary`.
3. Player labels below each score: Inter 900, 12px, letter-spacing 0.1em, `--text-secondary`. "P1" under winner, "P2" under loser (or left/right positionally).
4. Rematch button: unchanged `.rematch-btn` style.

Entrance animation: `fade-in-up 0.4s ease` at staggered delays (existing `@keyframes fade-in-up`).

Prohibited: "K.O." text, HP bars, damage stats, hit counts, round win counts. Only data shown: final scores and winner declaration.

### Modified: `useSpectatorSocket` (plan 09-01)

New state fields for dance:
- `gameType: 'boxing' | 'dance' | null` — set from `MsgJoined.game_type` on connect.
- `danceScores: [number, number]` — cumulative scores, updated on `dance_score` events.
- `danceBeat: { beat: number; totalBeats: number; targetPose: number[][] } | null` — current beat, updated on `dance_beat` events.
- `danceSnapshot` handling: consume spectator snapshot for mid-join score/beat state.

Game-type routing: `App.tsx` renders `<HudLayer>` for `gameType === 'boxing'` and `<DanceHud>` for `gameType === 'dance'`. Match end routes to boxing `.match-end-overlay` or dance match end layout accordingly.

### Modified: Mobile CalibrationOverlay (plan 09-04)

When `game_type === 'dance'` is received in `MsgJoined`:
- Skip the calibration phase entirely. `useGameSocket` phase transitions directly from `'lobby'` to `'match'` without going through `'calibration'`.
- `CalibrationOverlay` is never rendered.
- No visual changes to the calibration component itself — it is simply not shown.

## Interaction States

### Dance HUD — Beat Indicator Bar

| State | Visual |
|-------|--------|
| Beat just received | Hard snap to 100% width (transition: 0ms) |
| Between beats | Draining linearly to 0% over `beat_duration_ms` |
| No data yet (loading) | Bar at 0% width, beat label shows `— / —` |

### Dance Score Row

| State | Visual |
|-------|--------|
| Waiting (before first score) | Scores show `0.0` |
| Score update | Number updates immediately on `dance_score` event — no transition |
| Tie at match end | Both scores in `--text-secondary`, no "WINNER" label, show "TIED" |

### Target Pose Skeleton

| State | Visual |
|-------|--------|
| No beat received yet | Skeleton not rendered (Graphics object hidden) |
| Beat received | Fade in new skeleton over 150ms |
| Beat swap | 150ms fade-out → redraw → 150ms fade-in |
| `prefers-reduced-motion` | Fade transitions disabled; skeleton swaps instantly |

### Mobile — Dance Room Connection

| State | Visual |
|-------|--------|
| `game_type === 'dance'` in `MsgJoined` | Skip calibration; proceed directly to match |
| `game_type === 'boxing'` in `MsgJoined` | Calibration flow unchanged |
