## Component Inventory

### 1. Fast-Join View (new)

Triggered when `allParamsPrefilled === true` (all three of `?server=`, `?room=`, `?slot=` present in URL).

**Layout:** Full-screen centered column, max-width 480px, same as existing `.connection-screen`. No card/panel elevation — flat on `--bg-deep`.

**Structure (top to bottom):**

```
[SPECTRE title]          ← existing .title, unchanged
[metadata row]           ← NEW: "BOXING · ROOM ABC123 · PLAYER 1"
[spacer 32px]
[Join game button]       ← full-width primary button
[Enter manually link]    ← 16px below button, centered
```

**Metadata row spec:**
- Content: `{GAME_TYPE} · ROOM {CODE} · PLAYER {N}`
- If `?game=` param absent: `ROOM {CODE} · PLAYER {N}` (no game type, per D-03)
- Font: Inter 800, 12px, letter-spacing 0.10em, uppercase, `--text-primary`
- Separator `·` character: `--text-secondary`, same size and weight
- Display format: game type in uppercase (e.g. `BOXING`), room code in uppercase all-caps (e.g. `ABC123`), player as `PLAYER 1` or `PLAYER 2` (full word, not `P1` — more readable on mobile at a glance)
- Margin-bottom: 32px (xl)

**"Join game" button:**
- Uses existing `.big-button` class (Inter 800 1rem, min-height 52px, border-radius 4px)
- Background: `--accent` with 15% opacity: `color-mix(in oklch, var(--accent) 15%, transparent)` — matches lobby button pattern (more refined than solid `--accent`)
- Border: 1px `--accent`
- Hover: background `color-mix(in oklch, var(--accent) 25%, transparent)`, border `--accent-bright`
- Active: `transform: scale(0.97)` 80ms ease-out
- Loading state (connecting): label `Joining...`, button disabled, opacity 0.6
- Label: `Join game` — Inter 800 16px uppercase letter-spacing 0.08em

Rationale for `color-mix` background vs. solid `--accent`: Solid crimson background on mobile is aggressive for a "tap to confirm" action. The lobby landing page uses the same 15% tint for "Create Room" (enabled state) — Phase 5 matches that pattern for visual consistency.

**"Enter manually" link:**
- Position: centered below "Join game" button, margin-top 16px
- Text: `Enter manually`
- Font: Inter 400, 12px, NOT uppercase, letter-spacing 0.04em
- Color: `--text-secondary` at rest; `--text-primary` on hover/focus
- No underline at rest; `text-decoration: underline` on hover
- Min tap target: height 44px achieved by `padding: 16px 12px` (visual text is small, padding provides the target area)
- Behavior: one-way toggle — clicking reveals the full form in-place; no way to collapse back

### 2. Partial-Prefill Form View

Triggered when `?server=` and `?room=` are present but `?slot=` is absent (landing page join flow).

**Changes from current form:**
- Server URL field: hidden (not rendered) when `?server=` param is present
- Room code field: pre-typed, user can still edit
- Slot picker: visible and required — user must pick P1 or P2
- Connect button: label unchanged (`Connect`); enabled once slot is selected (it always is, default P1)
- "Enter manually" escape hatch: NOT shown (per D-09 — only fast-join needs it)

**No visual changes to existing form elements** — slot picker, room code field, and connect button retain current CSS.

### 3. Full Form View (manual entry / "Enter manually" expanded)

The existing form, all fields visible. Triggered by:
- No URL params at all (bare open)
- User taps "Enter manually" from fast-join view

**Changes:** Server URL field must be revealed when `?server=` param IS present but user tapped "Enter manually" (D-08: "Enter manually" reveals all hidden fields including server URL). The current `.field` style for the server URL input is unchanged.

**Expansion animation (Claude's discretion — D-07):**
- Expand from fast-join to full form: animate `max-height` on the newly revealed form block from 0 to its natural height over 200ms `ease-out`
- Do NOT animate `height` directly on layout elements (DESIGN.md rule: "Never animate `height`... Only `transform` and `opacity`")
- Correct approach: wrap the form fields in a container, use `opacity: 0 → 1` (150ms) plus `transform: translateY(-8px) → translateY(0)` (150ms ease-out-quart) for the reveal
- The fast-join metadata row fades out (`opacity: 1 → 0`) over 100ms before the form slides in
- Total transition: fast-join fades (100ms) → form appears (150ms). Sequential, not overlapping.

### 4. Error States (improved, per D-10 / D-11)

Error messages appear in the existing `.error-banner` element. Phase 5 changes the copy and, for server-unreachable, adds a Retry button inside the banner.

**Three error scenarios:**

**Room not found (WS close code 4004):**
- Banner text: `Room {CODE} not found. Check the code or ask the host.`
- No retry button.
- Banner style: existing `.error-banner` (unchanged).

**Slot taken / room full (WS close code 4000):**
- Banner text: `That slot is already taken. Ask the host to assign you a different player slot.`
- No retry button.
- Banner style: existing `.error-banner` (unchanged).

**Server unreachable (WS error event — no close code):**
- Banner text: `Can't reach the server. Check your connection and try again.`
- Retry button: rendered INSIDE `.error-banner`, below the text, margin-top 8px.
- Retry button style: NOT `.big-button`. Use a secondary/outline style:
  - Min-height 44px, border-radius 4px
  - Background: `--bg-surface`
  - Border: 1px `--text-dim`
  - Color: `--text-primary`
  - Font: Inter 700 14px uppercase letter-spacing 0.08em
  - Label: `Retry`
  - Hover: border → `--accent` 60% opacity, background → `--accent` 8% opacity
  - Active: `transform: scale(0.97)` 80ms ease-out
  - Width: auto (fits content), centered in banner via `display: block; margin: 8px auto 0`

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Fast-join idle | All 3 params present | Metadata row + "Join game" button + "Enter manually" link |
| Fast-join connecting | "Join game" tapped | Button label `Joining...`, button disabled (opacity 0.6) |
| Fast-join → full form | "Enter manually" tapped | Fast-join fades (100ms), form slides in (150ms translateY + opacity) |
| Partial-prefill idle | server + room params, no slot | Form with server hidden, room pre-filled, slot picker visible |
| Full form idle | No params / after "Enter manually" | All fields visible, Connect button |
| Full form connecting | Connect tapped | Button label `Connecting...`, button disabled |
| Error: room not found | WS close 4004 | Banner with copy, no retry |
| Error: slot taken | WS close 4000 | Banner with copy, no retry |
| Error: server unreachable | WS error event | Banner with copy + Retry button |
| Retry tapped | User taps Retry | Re-calls `onConnect` with cached (serverUrl, roomCode, slot) — no form re-entry |

## Layout Decisions (Claude's Discretion)

### Fast-join screen vertical rhythm

The fast-join screen is intentionally sparse — three elements: title, metadata row, button (+ escape link). This mirrors the "tight utility aesthetic" from Phase 4 room page (CONTEXT.md Specific Ideas).

Vertical distribution:
- Outer container: `justify-content: center` (existing `.connection-screen` behavior)
- Internal `gap: 16px` between elements by default
- Override: 32px gap between metadata row and button (use margin-top on button or dedicated gap override)

### "Enter manually" placement

Placed below the "Join game" button, centered, 16px below the button. Not in the header area, not in the footer. Rationale: players who scanned a QR are looking at the button — they won't notice the escape link, which is the intended behavior. Power users scanning for controls will find it just below.

### Room code display in fast-join header

- All uppercase (CSS `text-transform: uppercase` already applied to `.title` and `.subtitle` classes — apply same to this row)
- Letter-spacing 0.10em (matches `--type-label` from DESIGN.md)
- No special formatting box or pill around the code — plain text in the metadata row

### Player number display

`PLAYER 1` / `PLAYER 2` (full word). Rationale: on a 375px wide phone, the full metadata string `BOXING · ROOM ABC123 · PLAYER 1` is ~28 characters at 12px — well within one line. Full word is clearer than `P1` for first-time users.

## Responsive Behavior

This is a portrait-only mobile screen (landscape warning already in `app.css`). Target viewport: 320px–430px wide.

- `.connection-screen` max-width 480px centered — unchanged
- Metadata row: single line at all target viewports (≤28 chars at 12px). If it wraps (very narrow viewport or long room code — impossible since room codes are exactly 6 chars), allow natural wrap with `word-break: break-word`.
- "Join game" button: full-width (inherited from `.big-button` which stretches in the flex column)
- Error banner: full-width within `.connection-screen` padding, existing style
