# Phase 13.1 Plan 01: ml/ Pipeline Scaffold Summary

ml/ Python training pipeline scaffold created with requirements, documentation, and two runnable data-collection scripts.

## What Was Built

Two-task plan creating the complete offline half of the punch classifier pipeline:

**Task 1 — Directory scaffold and documentation:**
- `ml/data/.gitkeep` and `ml/models/.gitkeep` (empty dirs committed to repo)
- `ml/requirements.txt` with pinned Python dependencies (torch, onnxruntime, onnx, numpy, mediapipe==0.10.14, opencv-python, scikit-learn)
- `ml/scripts/__init__.py` (package marker)
- `ml/README.md` covering the full pipeline end-to-end: setup → data collection (BoxingVI Option A / webcam Option B) → extract → train → export → quantize → deploy to fps/public/models/
- `.gitignore` updated to exclude ml/data/**/*.mp4/.avi/.mov and ml/models/*.pt/*.onnx

**Task 2 — Data collection scripts:**
- `ml/scripts/extract_keypoints.py`: runs MediaPipe Pose on every .mp4/.avi/.mov under `--input-dir`, extracts 8 world landmark joints (JOINT_INDICES = [11,12,13,14,15,16,23,24]), saves sliding-window sequences as `.npy` files of shape `(window_size, 8, 3)`
- `ml/scripts/record_webcam.py`: guided webcam recording tool; opens camera, shows overlay prompts, records labeled .mp4 clips on SPACE keypress

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Lazy import pattern for mediapipe/cv2**
- **Found during:** Task 2 verification
- **Issue:** Both scripts had top-level `import cv2`, `import mediapipe` etc. The system Python 3.14.3 does not have these packages. `--help` and `--dry-run` verification commands would fail with ImportError before argparse even ran.
- **Fix:** In `extract_keypoints.py`, moved `cv2`/`numpy`/`mediapipe` into `_import_deps()` called only in the normal (non-dry-run) path. In `record_webcam.py`, `cv2` was already inside function bodies — no change needed there.
- **Files modified:** `ml/scripts/extract_keypoints.py`
