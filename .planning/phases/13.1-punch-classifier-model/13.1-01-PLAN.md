**ml/scripts/extract_keypoints.py**

CLI tool that runs MediaPipe PoseLandmarker on every .mp4/.avi/.mov file under `--input-dir`, extracts 8 world landmark joints per frame, and saves sliding-window sequences as .npy files to `--output-dir`.

Arguments:
- `--input-dir DIR` — root directory; expects subdirectories named by class label (jab/cross/hook_l/hook_r/guard)
- `--output-dir DIR` — where to write .npy files; same subdirectory structure as input
- `--window-size N` — frames per sequence, default 20 (per D3: T=20 at 30fps)
- `--stride N` — stride between windows during extraction, default 5 (per research: 50% overlap for data augmentation)
- `--dry-run` — print file count and exit without processing

Joint indices to extract (per D2 and research A2):
```python
JOINT_INDICES = [11, 12, 13, 14, 15, 16, 23, 24]
# [LEFT_SHOULDER, RIGHT_SHOULDER, LEFT_ELBOW, RIGHT_ELBOW,
#  LEFT_WRIST, RIGHT_WRIST, LEFT_HIP, RIGHT_HIP]
```

MediaPipe setup — use `mediapipe.solutions.pose.Pose` (compatible with 0.10.14):
```python
import mediapipe as mp
mp_pose = mp.solutions.pose
pose_detector = mp_pose.Pose(
    static_image_mode=False,
    model_complexity=1,
    min_detection_confidence=0.5,
    min_tracking_confidence=0.5,
)
```

Per-frame extraction: call `pose_detector.process(cv2.cvtColor(frame, cv2.COLOR_BGR2RGB))`. If `results.pose_world_landmarks` is None, skip frame. Extract the 8 joints as `[[lm.x, lm.y, lm.z] for i, lm in enumerate(results.pose_world_landmarks.landmark) if i in JOINT_INDICES]` — preserve order as defined in JOINT_INDICES.

Sliding window: accumulate extracted frames into a deque of length `window_size`. Once full, save the window as an .npy array of shape `(window_size, 8, 3)` using `np.save`. Advance by `stride` frames.

Output filenames: `<label>/<video_stem>_w{window_idx:04d}.npy`. Example: `jab/sparring01_w0000.npy`.

Print progress: one line per video with count of windows extracted.

Add `if __name__ == '__main__':` guard around argparse. `--dry-run` must print "DRY RUN: found N video files, would extract M windows" and exit with code 0.

**ml/scripts/record_webcam.py**

CLI tool for guided webcam data collection. Opens the default webcam, shows the live feed, records labeled clips.

Arguments:
- `--output-dir DIR` — where to save .mp4 files
- `--label LABEL` — one of: jab, cross, hook_l, hook_r, guard
- `--clip-duration SECONDS` — length of each clip, default 2.0
- `--clips N` — number of clips to record, default 50
- `--dry-run` — print config and exit without opening camera

Behavior:
- Display video feed in a cv2 window titled "Record: {label}"

- When SPACE pressed: show "RECORDING..." overlay for clip_duration seconds, then auto-stop
- Save each clip as `<output-dir>/<label>/<label>_{timestamp}.mp4` using `cv2.VideoWriter` with mp4v codec
- After all clips: print summary "Recorded N clips to <output-dir>/<label>/"
- 'q' key exits early

Print usage instructions to stdout at startup: what poses to perform for each class.
