pub struct RoomView { pub slots: [SlotView; 2], pub solo_mode: bool }

pub enum GameEvent {
    Hit { attacker: u8, defender: u8, region: BodyRegion, damage: f32, position: [f32; 2] },
    RoundOver { winner: Option<u8> },
    SendToPlayer { slot: u8, payload: Value },
    Broadcast { payload: Value },
    CommentaryHint { kind: String, payload: Value },
}

pub trait GamePlugin: Send + Sync {
    fn init_state(&self) -> Box<dyn Any + Send>;
    fn on_tick(&self, ctx: &TickContext, state: &mut dyn Any) -> Vec<GameEvent>;
    fn max_wins(&self) -> u32 { 2 }
    fn on_player_join(&self, _slot: u8, _state: &mut dyn Any) {}
    fn on_player_leave(&self, _slot: u8, _state: &mut dyn Any) {}
    fn on_calibration_complete(&self, _slot: u8, _ref_vel: f64, _state: &mut dyn Any) {}
    fn on_round_reset(&self, _state: &mut dyn Any) {}
}

    GamePlugin implementation — EXACT requirements:

    `init_state`: Return Box<DanceState> with scores=[0.0;2], target_index=0, round_start_tick=0,
    round_started=false, beats_scored=0, round_ended=false.

    `on_tick`:
    1. Downcast: `let s = state.downcast_mut::<DanceState>().expect("dance plugin: state type mismatch");`
    2. Early return if round_ended: `if s.round_ended { return vec![]; }`
    3. Lock round_start_tick on first tick (Pitfall 1):
       ```rust
       if !s.round_started {
           s.round_start_tick = ctx.tick_info.tick;
           s.round_started = true;
       }
       ```
    4. Beat clock using saturating_sub (avoids overflow if tick < round_start_tick):
       ```rust
       let elapsed_ticks = ctx.tick_info.tick.saturating_sub(s.round_start_tick);
       ```
    5. Beat boundary: fires when `elapsed_ticks > 0 && elapsed_ticks % BEAT_INTERVAL == 0 && s.beats_scored < TOTAL_BEATS`
    6. At beat boundary:
       - Get target: `let target = &poses::POSE_LIBRARY[s.target_index % poses::POSE_LIBRARY.len()];`
       - Score each slot: for slot_idx in 0..2, skip slot 1 if ctx.room.solo_mode; use `ctx.frames[slot_idx].back()` for the most recent frame (last frame before beat transition)
       - `s.scores[slot_idx] += score_pose(frame, &target.keypoints);`
       - Increment beats_scored and advance target_index
       - Push two Broadcast events:
         ```rust
         GameEvent::Broadcast { payload: json!({
             "type": "dance_beat",
             "beat": s.beats_scored,
             "total_beats": TOTAL_BEATS,
             "target_pose": target.keypoints.iter()
                 .map(|kp| [kp.x, kp.y, kp.z, kp.visibility])
                 .collect::<Vec<_>>(),
         })}
         GameEvent::Broadcast { payload: json!({
             "type": "dance_score",
             "beat": s.beats_scored,
             "scores": [s.scores[0], s.scores[1]],
         })}
         ```
    7. Round end check (AFTER beat scoring, so beat 16 is scored before RoundOver):
       ```rust
       if s.beats_scored >= TOTAL_BEATS {
           s.round_ended = true;
           let winner = if ctx.room.solo_mode {
               Some(1u8)  // solo: player 1 always "wins"
           } else if s.scores[0] > s.scores[1] {
               Some(1)
           } else if s.scores[1] > s.scores[0] {
               Some(2)
           } else {
               None  // draw
           };
           events.push(GameEvent::RoundOver { winner });
       }
       ```

    `on_round_reset`:
    - Downcast
    - Reset: scores=[0.0;2], beats_scored=0, target_index=0, round_started=false, round_ended=false
    - Do NOT reset round_start_tick (it will be re-captured via round_started=false on next first tick)

    `on_calibration_complete`: intentional no-op (D-05). Provide default no-op inherited from trait.
    Override ONLY if you need to confirm it's truly a no-op for documentation purposes — use

    Free function `score_pose(player_frame: &PoseFrame, target: &[PoseKeypoint]) -> f64`:
