<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → POST /rooms | Untrusted `?game=` query parameter crosses the HTTP boundary into server |
| HTTP client → GET / | Static HTML response; no user input parsed |
| WebSocket client → ws_player | room_code path param from untrusted client; Option A removes implicit room creation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-P2-01 | Tampering | POST /rooms `?game=` param | mitigate | Axum `Query<CreateRoomParams>` deserializes via serde; unknown values produce 400 BAD_REQUEST with error JSON before any room creation; validated by registry lookup (`app.plugins.get(game)`) |
| T-03-P2-02 | Denial of Service | POST /rooms rapid room creation | accept | Out of scope per REQUIREMENTS.md Out-of-Scope (no rate limiting, no auth); existing 10-minute room expiry task (room_manager.rs expiry_task) bounds accumulation |
| T-03-P2-03 | Spoofing | ws_player room_code path param pointing to non-existent room | mitigate | Option A: ws_player returns early if room not found (no implicit room creation); attacker cannot force room creation via WS path |
| T-03-P2-04 | Information Disclosure | POST /rooms error response leaks game name | accept | Error message `unknown game: {game_name}` echoes user input; game names are not secrets; consistent with existing log-and-continue pattern |
| T-03-P2-05 | Denial of Service | LOBBY_HTML served as const &str | accept | No filesystem read, no external resource; response is a fixed-size string; no amplification vector |
| T-03-P2-06 | Elevation of Privilege | AppState.plugins HashMap mutation after Arc wrap | mitigate | HashMap built before Arc::new — immutable after wrap; no runtime mutation path exists (Pitfall 2 in RESEARCH.md) |
</threat_model>
