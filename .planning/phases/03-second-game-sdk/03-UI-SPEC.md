# Phase 3 — UI Design Contract

> Visual and interaction contract for the static lobby HTML page delivered in Phase 3.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope

Phase 3 has exactly one new UI surface: a static HTML page served at `GET /` from the
Axum server, embedded as a `const &str` in `engine-core/src/main.rs`. No React, no build
step, no design system framework. The mobile TypeScript client and the spectator overlay
are **unchanged** — they are out of scope for this phase.

The lobby page lets a user select a game, call `POST /rooms?game=X`, and read back the
6-char room code to type into the mobile client.

Phase 3 introduces no reusable React components. The lobby is a single embedded HTML
string. The executor must implement these interactive elements directly in the HTML:

| Element | HTML Semantics | CSS Pattern Source |

| Game picker — Boxing button | `<button onclick="createRoom('boxing')">` | `.big-button` pattern from app.css:136 |

token (16px); horizontal padding uses the `lg` token (24px). Touch target minimum 44px
(accessibility minimum) — both buttons exceed this at 52px.

Button layout: two buttons side by side with `display: flex; gap: 8px` — matches
`.mode-toggle` pattern in app.css:179.

Button state:
- Default: `background: var(--bg-surface); border: 1px solid var(--text-dim); color: var(--text-primary)`
- Hover: `background: var(--accent); border-color: var(--accent-bright)`
- Active (pressed): `transform: scale(0.98)` — matches `app.css .big-button:active`
- Loading (after click, before response): button `disabled` attribute set; opacity 0.5

## Interaction Contract

### Game Selection and Room Creation Flow

1. User lands on `GET /` — sees the heading and two game buttons (Boxing, Dance).
2. User clicks a button — button receives `disabled` attribute immediately (prevent double-submit).
3. `fetch('/rooms?game={game}', { method: 'POST' })` fires.
4. On 201 response with `{ "room_code": "XYZ123" }` — display `data.room_code` in
   `#room-code`. Re-enable buttons so user can create another room.
5. On non-ok response — display `data.error ?? 'Server error'` in `#room-code` using
   the error color (`#ff9b9b`). Re-enable buttons.
6. On network failure (fetch throws) — display `'Could not reach server'` in `#room-code`
   using error color. Re-enable buttons.

No full-page reload, no form submission. JavaScript is inline in the `<script>` block.
No external script sources. The `createRoom` function is the only JS function defined.

### Touch Target Compliance

All interactive buttons: `min-height: 52px` — exceeds WCAG 2.5.5 recommended 44px minimum.
The page is desktop-oriented (server admins, game operators) but must not break on mobile.
`<meta name="viewport" content="width=device-width, initial-scale=1.0">` is required.

## Page Structure (HTML Skeleton)

The executor MUST follow this structure when embedding the HTML string in `main.rs`.
All CSS is inline in the `<style>` block — no external stylesheets, no CDN links.

```
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>PoseEngine Lobby</title>
  <style>
    :root { /* token declarations — see Token Alignment table */ }
    body { /* bg-deep background, text-primary color, font stack; max-width: 480px; margin: 48px auto; padding: 0 16px */ }
    h1 { /* 28px, weight 800, letter-spacing 0.12em, uppercase, text-primary */ }
    p.subtitle { /* 13px, weight 800, text-secondary, uppercase, letter-spacing 0.08em */ }
    .btn-row { /* flex, gap 8px */ }
    button { /* big-button pattern: min-height 52px; padding: 16px 24px; accent bg on hover, text-primary */ }
    #room-code { /* 32px, weight 800, letter-spacing 0.2em, margin-top 16px */ }
    #room-code.error { /* #ff9b9b color, error-banner background/border */ }
  </style>
</head>
<body>
  <h1>Choose a Game</h1>
  <p class="subtitle">Create a room, then enter the code in the mobile app</p>
  <div class="btn-row">
    <button onclick="createRoom('boxing')">Boxing</button>
    <button onclick="createRoom('dance')">Dance</button>
  </div>
  <div id="room-code"></div>
  <script>
    async function createRoom(game) { /* POST /rooms?game=game, display result */ }
  </script>
</body>
</html>
```

Source: RESEARCH.md Pattern 5 (verified against axum 0.8.9 `axum::response::Html`).
