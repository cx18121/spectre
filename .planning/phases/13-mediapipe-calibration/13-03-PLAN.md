<objective>
Wire the pose pipeline into App.tsx and add phase-driven routing for calibration and match screens (WCI-01, WCI-02, WCI-04 completion).

This plan extends App.tsx with four additions:
1. `usePose(videoRef, cameraReady, warmup.workerRef)` — called unconditionally at App level (not inside a screen component) so Phase 14 can consume the same keypoints without a second detection loop.
2. `useOneEuroFilter(pose.keypoints)` — smooths raw worldLandmarks before downstream consumers.
3. `CalibrationScreen` rendered when `socket.phase === 'calibration'`, receiving stream + smoothed keypoints + onCalibrationDone.
4. `onCalibrationDone` sends `{ type: 'calibration_done', reference_velocity: refVel }` via `socket.send` — exactly matching `MsgCalibrationDone` from shared/protocol.ts.
5. Game canvas root div rendered when `socket.phase === 'match'` (Phase 14 hook point).

Purpose: Connects all Phase 13 hooks to the live application and completes the WCI-04 handshake — after `MsgCalibrationDone` is sent, the server emits `MsgMatchStart` (handled by useGameSocket) which advances `socket.phase` to 'match'.

Output: Updated App.tsx + tests. No other files modified.
</objective>
