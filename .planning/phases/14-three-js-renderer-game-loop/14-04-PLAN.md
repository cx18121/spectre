<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MsgFpsState → HP display | Server-supplied HP values drive HP bar widths; attacker could send malicious HP values |
| matchEnd.winner → WIN/LOSE display | Server decides winner; client only renders based on server decision |
| onRematch → socket.setPhase | Rematch triggers client phase change; no network message sent until CalibrationScreen completes calibration |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-04-01 | Tampering | HP bar width from server-supplied hp values | mitigate | Clamp HP before computing percentage: `const pct = Math.max(0, Math.min(100, (hp / 800) * 100))`. Prevents negative-width or >100% bars from malformed server data. |
| T-14-04-02 | Tampering | matchEnd.winner determines WIN/LOSE display | accept | Server is authoritative for winner determination. Client display only. No game logic (damage, scoring) is computed client-side from this value. |
| T-14-04-03 | Denial of Service | round_timer NaN/Infinity from server | mitigate | Display: `const display = isFinite(roundTimer) && roundTimer >= 0 ? Math.ceil(roundTimer) : 0`. Prevents NaN from rendering in the timer pill. |
| T-14-04-04 | Spoofing | Rematch initiates new calibration | accept | Rematch only changes local client phase to 'calibration'. No server room ownership change occurs. Server handles the new calibration_done message when the player completes it — same security boundary as initial match start. |
</threat_model>
